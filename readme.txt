Описание установки и запуска:

1. С помощью docker:
   1) Перейти в папку с проектом.

   2) Собрать проект с помощью команды:
       mvn clean package

   3) Создать образ и запустить
   docker build -f Dockerfile -t online-test .
   docker run -p 8080:8080 online-test


2. Без docker:
   1) Перейти в папку с проектом.

   2) Собрать проект с помощью команды:
       mvn clean package

   3) Перейти в папку target и запустить приложение:
      cd target
      java -jar onlineTest.jar


Для тестирования можно воспользоваться браузером по ссылке: http://localhost:8080


Описание сервисов (тип авторизации Basic Auth)

1. /queries  (GET)  - Получить список всех вопросов с вариантами ответов, (авторизоваться любым пользователем, например логин:u, пароль:1)

2. /queries  (POST)  - добавить вопрос, авторизация админ (Логин:admin, пароль:1)

3. /users (GET) - получить список пользователей, авторизация админ (Логин:admin, пароль:1)

4. /users (POST) - добавить пользователя (без авторизации)
 
5. /answers (GET) - Получить все вопросы\ответы текущего пользователя (авторизоваться любым пользователем)

6. /answers (POST) - Отправить на проверку все ответы на вопросы пользователя (авторизоваться любым пользователем)

7. Статистика:  Все GET.
   
Необходимо авторизоваться администратором (Логин:admin, пароль:1)
   /countUsers - Сколько всего пользователей зарегистрировано в системе
   
   /countUsersTested - Сколько пользователей прошли тестирование
   
   /countUsersTestedAllQuestions - Сколько пользователей ответили на все вопросы тестирования
   
   /countUsersTestedAllQuestionsSuccess - Сколько пользователей ответили на все вопросы тестирования правильно
   
Необходимо авторизоваться пользователем (Логин:u, пароль:1) или вновь заведенным.
   /countRightAnswers - Процент прохождения тестирования текущего пользователя (сколько правильных ответов он дал)
   
   /countUsersTestedWorseCurrent - Сколько процентов пользователей справилось с тестированием хуже текущего пользователя
   
   /countUsersTestedBetterCurrent - Сколько процентов пользователей справилось с тестированием лучше текущего пользователя
   
   /countUsersTestedEqualCurrent - Сколько процентов пользователей справилось с тестированием также как и текущий пользователь
